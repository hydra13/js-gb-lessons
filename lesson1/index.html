4. Напишите, являются ли данные имена переменных корректными:
let mode = "normal";
let my_valu3 = 102;
let 3my_value3 = "102";
let __hello__ = "world";
let $$$ = "money";
let !0_world = true;

Решение:
let mode = "normal"; - является корректным

let my_valu3 = 102; - является корректным, но лучше использовать CamelCase - myValu3 и, в данном случае, использовать полное наименование (либо myValue3, либо myValue)

let 3my_value3 = "102"; - является некорректным, т.к. название переменных не может начинаться с цифр

let __hello__ = "world"; - является корректным, т.к. в JavaScript позволяется наименование переменных начинать со знаков нижнего подчеркивания и доллара, но в данном случае лучше использовать просто hello в качестве имени переменной

let $$$ = "money"; - является корректным, т.к. в JavaScript позволяется наименование переменных начинать со знаков нижнего подчеркивания и доллара, но в данном случае лучше в качестве наименования переменной использовать значение, которое будет отображать суть переменной

let !0_world = true; - является некорректным, т.к. название переменных не может начинаться со специальных символов (исключение - _ и $)
